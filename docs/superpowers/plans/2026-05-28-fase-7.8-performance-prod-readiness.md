## Task 1: Reuse Pre-Compiled RegExp in scanPatterns()

**Files:**
- Modify: `src/dlp/patterns.ts:331-332`

**Context:** `scanPatterns()` creates `new RegExp(def.pattern.source, def.pattern.flags)` on every call. Each PATTERNS entry already has a compiled `RegExp` object as `def.pattern`. The only reason to recreate would be to reset `lastIndex` — but that can be done with `re.lastIndex = 0` in place, without recompilation. With ~30 patterns and calls triggered on every 400ms debounce tick, this saves 30 RegExp compilations per scan.


## Task 3: ResizeObserver — Remove documentElement + RAF Debounce

**Files:**
- Modify: `src/content/injectButton.ts:537-543`

**Context:** The ResizeObserver observes 3 targets:
```typescript
ro.observe(input);
ro.observe(findVisualContainer(input));
ro.observe(document.documentElement);
```

`document.documentElement` fires on every DOM mutation in the page (ChatGPT, Claude re-render their UIs constantly), triggering `update()` on every keystroke. The window already has a `resize` event listener (line ~546: `window.addEventListener('resize', update)`) that handles viewport changes. `document.documentElement` is redundant and expensive.

Additionally, `update()` runs synchronously on every ResizeObserver callback tick. Batching with `requestAnimationFrame` limits it to once per frame.

Current lines 537-543:
```typescript
  let ro: ResizeObserver | undefined;
  if (typeof ResizeObserver !== 'undefined') {
    ro = new ResizeObserver(update);
    ro.observe(input); // track textarea/div growth directly
    ro.observe(findVisualContainer(input));
    ro.observe(document.documentElement);
  }
```

- [ ] **Step 1: Apply the fix**

Replace lines 537-543 with:

```typescript
  let ro: ResizeObserver | undefined;
  if (typeof ResizeObserver !== 'undefined') {
    let rafId: number | undefined;
    ro = new ResizeObserver(() => {
      if (rafId !== undefined) cancelAnimationFrame(rafId);
      rafId = requestAnimationFrame(() => { rafId = undefined; update(); });
    });
    ro.observe(input);
    ro.observe(findVisualContainer(input));
    // document.documentElement removed — window 'resize' listener already covers viewport changes
  }
```

- [ ] **Step 3: Apply the timeout fix**

In `src/dlp/imageInterceptor.ts`, replace `handleImageFile` (lines 39-54) with:

```typescript
async function handleImageFile(file: File): Promise<void> {
  if (!file.type.startsWith('image/')) return;
  const token = await getToken();
  if (!token) return;
  const base64 = await fileToBase64(file);

  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), 8000);

  try {
    const resp = await fetch(`${BFF_BASE}/dlp/image`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${token}` },
      body: JSON.stringify({ image_b64: base64 }),
      signal: controller.signal,
    });
    clearTimeout(timeoutId);
    if (!resp.ok) return;
    const data = await resp.json() as { show_warning: boolean; advisory?: string; entities?: Array<{ type: string }> };
    if (data.show_warning) {
      showImageDlpBanner(data.advisory || data.entities?.[0]?.type || 'PII');
    }
  } catch (err) {
    clearTimeout(timeoutId);
    if ((err as Error).name === 'AbortError') return; // timeout — fail open, don't block user
    throw err;
  }
}
```
