# FASE 7.9 — modal.ts God Object Refactoring

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split `src/ui/modal.ts` (3490 lines, God Object) into focused modules under `src/ui/modal/`, preserving all behaviour, all exports, and all tests.

**Architecture:** Use barrel strategy — `src/ui/modal.ts` becomes `export * from './modal/index'` so all existing importers (`injectButton.ts`, `content.ts`, `popup.ts`) continue to work without modification. A new `src/ui/modal/index.ts` re-exports the public API from 10 focused sub-modules. Mutable module-level state (`promptCache`, `upgradeShown`, `msgIntervalId`) moves to a shared `state.ts` object to avoid cyclic imports.

**Tech Stack:** TypeScript, Vite IIFE build (`inlineDynamicImports: true` — dynamic import NOT available), Vitest (272 tests baseline), Playwright E2E (T1–T6).

## CRITICAL: How Each Task Works

Every task follows the same 5-step mechanical pattern:

1. **Create new file** at `src/ui/modal/<name>.ts` — copy the specified functions from `modal.ts`, keeping **exact implementations unchanged**. Add only the imports those functions actually need.
2. **Delete those functions** from `src/ui/modal.ts` (the monolith).
3. **Add an import** at the top of `src/ui/modal.ts`: `import { ... } from './modal/<name>';`
4. Run `npm run build` — fix any TypeScript errors (usually missing imports in the new file).
5. Run `npx vitest run` — must show **272 passed, 0 failed**.

**Never change function signatures, logic, or exports.**
