## 🔒 Security & Compliance

### LGPD Compliance Checklist

- ✅ **Zero PII Storage:**
  - Entity types stored (e.g., "BR_CPF")
  - Never values (e.g., never "050.423.674-11")
  - Payload correlation via SHA-256 hash

- ✅ **User Consent:**
  - Badge + warnings shown before send
  - Override option for Free users
  - Server logs all decisions

- ✅ **Exception Safety:**
  - PII redacted from error logs
  - Stack traces sanitized
  - No sensitive data in responses

- ✅ **Data Retention:**
  - Expires_at field ready
  - TTL policies configurable
  - No indefinite storage

- ✅ **User Rights:**
  - RLS policies (users see own events)
  - Analytics exported via safe queries
  - Audit trail complete

### Security Layers

```
Layer 1: Frontend
├─ Local scanner (fast detection)
├─ Visual feedback (badges, banners)
└─ User override (with warnings)

Layer 2: Network
├─ HTTPS only
├─ JWT validation
└─ CORS restrictions

Layer 3: Backend
├─ Server-side revalidation
├─ Strict mode enforcement
├─ Client-server mismatch detection
└─ Timeout protection

Layer 4: Persistence
├─ Entity types only (never values)
├─ Hash correlation (no content)
├─ Exception sanitization
└─ Safe aggregates (no individuals)

Layer 5: Fallbacks
├─ In-memory cache if DB fails
├─ Safe defaults if engine times out
├─ Graceful degradation
└─ No service blocking
```

## 📦 Architecture

### Frontend (React Extension)

```
src/
├─ components/
│  ├─ DLPBadge.tsx       (HIGH/MEDIUM/LOW indicator)
│  ├─ ProtectModal.tsx   (Rewrite UI)
│  └─ SettingsPanel.tsx  (Plan + DLP config)
├─ services/
│  ├─ scanner.ts         (Presidio local)
│  ├─ api.ts             (Backend calls)
│  └─ storage.ts         (chrome.storage.local)
└─ hooks/
   ├─ useDLP.ts          (DLP state management)
   └─ useAuth.ts         (JWT + plan)
```

### Backend (FastAPI + Presidio)

```
backend/
├─ dlp/
│  ├─ engine.py          (Server analyzer)
│  ├─ entities.py        (Custom recognizers)
│  ├─ pipeline.py        (Full scan)
│  ├─ enforcement.py     (Strict mode)
│  ├─ telemetry.py       (Event emission)
│  ├─ analytics.py       (Safe queries)
│  └─ supabase_telemetry.py (Persistence)
├─ routes/
│  ├─ dlp.py            (POST /dlp/scan)
│  ├─ analytics.py      (GET /analytics/...)
│  └─ auth.py           (JWT validation)
└─ middleware/
   └─ exception_sanitizer.py (PII redaction)
```

### Database (Supabase PostgreSQL)

```
dlp_events table:
├─ id (uuid primary key)
├─ user_id (uuid nullable)
├─ event_type (text)
├─ risk_level (enum: CRITICAL/HIGH/MEDIUM/LOW/SAFE)
├─ entity_types (text[]) ← Types only!
├─ entity_count (int)
├─ was_rewritten (bool)
├─ timeout_occurred (bool)
├─ error_occurred (bool)
├─ duration_ms (int)
├─ score (float)
├─ provider (text)
├─ endpoint (text)
├─ session_id (uuid)
├─ hashed_payload_id (text) ← Hash, never content
├─ created_at (timestamptz)
├─ expires_at (timestamptz)
└─ indexes: user_created, risk_level, entity_types (GIN), session, event_type
```

## 🔄 Data Flow Example

```
User types: "CPF 050.423.674-11"
          ↓
[Frontend] Local scanner detects BR_CPF
          ↓
Badge shows "HIGH" (red)
          ↓
User clicks "Protect Data"
          ↓
Rewrite: "CPF [CPF]"
          ↓
Send POST /generate-prompts:
{
  "input": "CPF [CPF]",
  "dlp": {
    "dlp_risk_level": "HIGH",
    "dlp_entity_types": ["BR_CPF"],
    "dlp_was_rewritten": true
  }
}
          ↓
[Backend] Server revalidates
          - Detects: HIGH (BR_CPF) ✓
          - Matches client? YES ✓
          - Strict mode? Check
          - Final payload: "[CPF]"
          ↓
[Telemetry] Log event:
{
  "event_type": "dlp_scan_complete",
  "risk_level": "HIGH",
  "entity_types": ["BR_CPF"],
  "was_rewritten": true,
  "duration_ms": 45
}
          ↓
[Gemini API] Receive: "CPF [CPF]..."
         NEVER: "050.423.674-11"
          ↓
Response: 3 prompt versions
          ↓
[Frontend] Display results
          ↓
User happy, data safe ✅
```
